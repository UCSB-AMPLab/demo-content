Markdown es un lenguaje de marcado ligero creado en 2004 por [John Gruber](https://daringfireball.net/) con la idea de que el formato en texto plano sea legible tal cual, incluso antes de convertirlo a HTML. A diferencia de los procesadores de texto que ocultan los códigos de formato, Markdown usa una sintaxis sencilla e intuitiva: asteriscos para énfasis (`*cursiva*` o `**negrita**`), numerales para encabezados (`# Encabezado`), corchetes para enlaces (`[texto](url)`) y guiones para listas.

El objetivo original era que escribir para la web fuera tan fácil como escribir un correo. De allí surgieron varios "sabores" de Markdown a lo largo del tiempo: [CommonMark](https://commonmark.org/) (un estándar con especificación rigurosa), [GitHub Flavored Markdown](https://github.github.com/gfm/) (que agrega tablas y listas de tareas), entre otros. A pesar de las variaciones, comparten la misma sintaxis central y siguen siendo legibles como texto plano.

Telar usa [Python-Markdown](https://python-markdown.github.io/) con varias extensiones activadas: notas al pie para citas académicas, tablas para datos estructurados y más. Esto significa que puedes escribir el contenido de los paneles en un editor de texto sencillo usando convenciones de texto plano conocidas.
